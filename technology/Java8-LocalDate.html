概念

  • UTC 指的是 Coordinated Universal Time - 世界协调时间, 是根据原子钟来计算时间，目前世界上最精确的原子钟50亿年才会误差1秒可以说非常精确。
  • GMT 指的是 Greenwich Mean Time - 格林威治时间, 是根据地球的转动来计算时间的，以英国伦敦郊区皇家格林威治天文台的标准时间。世界被分为24个时区, 而北京在东八区

Java Date 使用的是 UTC时间, UTC时间是用距离一个固定时间点的毫秒数 (可正可负) 表示的,这个点就是所谓的纪元,它是UTC时间 1970年1月1日 00:00:00.

但是,Date类提供的日期并没有太大的用途, 类库设计者决定将保存时间与给定时间点命名分开. 因为标准 Java类库分别包含了两个类: 一个是用来表示时间点的 Date类; 另一个是大家熟悉的日历表示法的 LocalDate类.

因此 Java 8中新增了LocalDate、LocalTime以及包含两者的LocalDateTime。

java.util.Date 和 SimpleDateFormatter 都不是线程安全的, 而这个新增加的LocalDate 和 LocalTime 以及LocalDateTime和最基本的String 一样是不变类型,不但是线程安全的, 而且不能修改.
